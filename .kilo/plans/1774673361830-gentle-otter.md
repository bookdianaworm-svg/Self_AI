## 1. Overview

### 1.1 Purpose

This plan details the implementation of a **Type Checking System** for the RLM verification stack. The system provides formal type verification for Haskell and Lean 4 code, enabling:
- Compile-time unit/dimension checking (Haskell)
- Theorem verification against Layer 1 axioms (Lean)
- Integration with the Slow Loop (verification phase of Dual-Loop Architecture)

### 1.2 Scope

| Include | Exclude |
|---------|---------|
| Haskell type checking via GHC | Haskell dimensional type library development |
| Lean verification via Lake | Full Lean kernel/IDE integration |
| Type checker abstraction layer | Cross-Domain Synthesis (Phase 4) |
| Integration with LocalREPL | Empirical Fuzzing Loop (Phase 5) |
| Redux state management | Advanced Edge Domains (Phase 6) |

### 1.3 Integration Point with Main Plan

The Type Checking System should be implemented **before Phase 2 (Dual-Loop Architecture)** or **concurrent with Phase 2, Sprint 1**, because:

1. The Slow Loop (Phase 2) requires type checking for verification
2. Domain Routing (Phase 3) needs type checking to classify tasks
3. Type checkers provide the "verification oracle" that the Dual-Loop references

## 7. Success Criteria

### 7.1 Functional Criteria

- [ ] `check_haskell_types("x :: Int; x = 'hello'")` returns type error with correct line/column
- [ ] `verify_lean("theorem test : 1 = 2 := rfl")` returns verification failure
- [ ] Health checks return correct availability status
- [ ] Type checking respects configured timeouts

### 7.2 Integration Criteria

- [ ] Type checkers work via LocalREPL
- [ ] Redux state reflects type checker status
- [ ] Verification agents can call type checkers
- [ ] Configuration loads from YAML

### 7.3 Quality Criteria

- [ ] 80% code coverage for type checking module
- [ ] All existing tests pass
- [ ] Documentation complete

## 8. Out of Scope (Future Phases)

The following are explicitly excluded from this plan and should be addressed in later phases:

1. **Dimensional Type Library**: Custom dimensional types in Haskell (Phase 4+)
2. **LeanDojo Integration**: Automated proof synthesis (Phase 4+)
3. **Cross-Domain Synthesis**: Multi-domain verification (Phase 4)
4. **Empirical Fuzzing**: Automata learning-based discovery (Phase 5)
5. **Async Queue Integration**: Moving to Slow Loop (Phase 2)
6. **Bounce-Back Interrupt**: Error translation for Fast Loop (Phase 2)
